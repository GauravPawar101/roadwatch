# Ministry-level PDF report format

The ministry report is generated by the gateway API endpoint:

- `GET /reports/ministry.pdf` (requires `CE` role)

This is a single PDF intended for ministry/department review across districts.

## Sections

1) **Header**
- Title: `RoadWatch Ministry-Level Report`
- Generated timestamp
- Period (optional query params `from`/`to` are currently informational)

2) **National Summary**
- Total complaints
- Totals by status (e.g. `PENDING`, `IN_PROGRESS`, `ESCALATED`, `RESOLVED`, `REJECTED`)
- Chronic count: unresolved complaints older than `chronicDays` (default 60)

3) **District Breakdown**
- One-line rows per district:
  - Total complaints
  - Unresolved count
  - Resolved count
  - Escalated count

4) **Hotspots**
- Top geo clusters (grid-based, 30-day default in the current implementation)
- Shows cluster key, complaint count, centroid lat/lng

5) **Contractor Performance (scorecard basis)**
- Top contractors by resolved/assigned volume
- Shows: assigned, resolved, open, average resolution duration, on-time rate

## Notes

- The PDF uses `pdfkit` and is generated server-side in `apps/gateway-api/src/reports/ministryPdf.ts`.
- Contractor scorecards require complaints to be assigned using `POST /authority/complaints/:id/assign`.
- Chronic rule matches the public chronic feed: `status <> 'RESOLVED'` and age >= `chronicDays`.
